Lattice Documentation

Technical reference for the Lattice content-addressable storage and peer-to-peer networking stack. Built in Swift 6 for Apple and Linux platforms.

The Ecosystem

Lattice is a modular stack of Swift packages that together form a complete peer-to-peer content-addressable storage system. Each package has a single responsibility and can be used independently or composed into a full node.

Design Principles

Actor-Based Concurrency

Every storage worker is a Swift actor, providing compile-time data race safety. The AcornCASWorker protocol requires actor conformance, ensuring all implementations are thread-safe by construction.

Content Addressing

All data is identified by the SHA-256 hash of its contents. This makes storage inherently deduplicating, verifiable, and cacheable. A ContentIdentifier computed from data in Tokyo will be identical to one computed from the same data in London.

Worker Chaining

Workers link together via near and far references, forming a storage hierarchy. The protocol's default get() implementation walks the chain from fast to slow, backfilling faster layers on hits. No orchestration code needed — the chain self-optimizes.

Cross-Platform

All packages compile on both Apple platforms (macOS, iOS, tvOS, watchOS, visionOS) and Linux. Platform-specific APIs are conditionally compiled — OSAllocatedUnfairLock on Apple, NSLock on Linux; Network.framework discovery wrapped in #if canImport(Network).
