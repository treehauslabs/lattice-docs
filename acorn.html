VolumeBroker Core

The foundation of Lattice storage. Defines the Volume-granular storage protocol, Volume payloads (root + entries), ref-counted pin lifecycle with TTL and owner tracking, and BrokerFetcher/BrokerStorer adapters for Cashew integration.

.package(url: "https://github.com/treehauslabs/VolumeBroker.git", from: "1.0.0")

VolumeBroker Protocol

The core protocol that all storage brokers implement. Requires actor conformance for thread safety. Storage is Volume-granular: a Volume is a root CID together with all of its constituent entry CIDs, stored and evicted as a unit.

public protocol VolumeBroker: Actor {
    func hasVolume(root: String) async -> Bool
    func fetchVolumeLocal(root: String) async -> Volume?
    func storeVolumeLocal(_ volume: Volume) async

    func pin(root: String, owner: String, count: Int, ttl: Duration?) async
    func unpin(root: String, owner: String, count: Int) async
    func unpinAll(owner: String) async
    var owners: Set<String> { get async }

    func evictUnpinned() async
}

Required Methods

MethodDescription
hasVolume(root:)Check if this broker has a Volume with the given root CID locally.
fetchVolumeLocal(root:)Retrieve a Volume (root + all entries) from this broker's local storage.
storeVolumeLocal(_:)Store a Volume in this broker's local storage. The Volume is stored as a unit.
pin(root:owner:count:ttl:)Add count ref-counted pins for the given owner. Optional TTL sets automatic expiry.
unpin(root:owner:count:)Decrement pin count for the given owner by count. Pins at zero are removed.
unpinAll(owner:)Remove all pins held by the given owner across all Volumes.
ownersThe set of all owners that currently hold pins.
evictUnpinned()Remove all Volumes that have no active pins. Pinned Volumes are never evicted.
Volume-granular, not CID-granular
Unlike per-CID storage, VolumeBroker stores and evicts entire Volumes as units. A Volume groups a root CID with all the entry CIDs it references. This ensures that partial eviction never leaves orphaned entries or dangling root references.

Volume Payload

A Volume is the fundamental unit of storage. It pairs a root CID with a dictionary of entry CIDs to their data. All entries belong to the root and are stored, fetched, and evicted together.

public struct Volume: Sendable {
    public let root: String                    // root CID
    public let entries: [String: Data]         // CID → data for all entries

    public init(root: String, entries: [String: Data])
}

Volume Structure

Each Volume represents a self-contained unit of content-addressed data:

  • root: The root CID that identifies this Volume. Typically the root of a Cashew Merkle subtree.
  • entries: A flat dictionary mapping each entry CID to its serialized data. Includes the root's own data as well as all referenced child nodes.

When a Cashew Merkle tree mutation produces a new root, the changed nodes form a new Volume. Unchanged subtrees remain in their existing Volumes, providing natural deduplication.

Pin Lifecycle

Pins prevent Volumes from being evicted. They are ref-counted per owner and optionally time-limited via TTL.

public struct PinEntry: Sendable {
    public let owner: String
    public var count: Int
    public var expiresAt: Date?
}

Pin Operations

  1. Pin on create: When a chain produces a new block, it pins the frontier state Volume with owner: chainID and count: 1.
  2. Accumulate: Subsequent pins from the same owner increment the count. A Volume with count: 3 requires three unpin calls to fully release.
  3. TTL expiry: If a TTL is provided, the pin automatically expires after the given duration. Expired pins are treated as absent during eviction checks.
  4. Unpin replaced roots: When a chain advances past its retention depth, it unpins the old state root Volume. Once all owners have released their pins, the Volume becomes eligible for eviction.
  5. Owner cleanup: unpinAll(owner:) removes all pins for a given owner across all Volumes. Used when a chain is removed or a node shuts down.

Eviction Rules

  • A Volume is pinned if it has at least one non-expired pin entry with count > 0.
  • evictUnpinned() removes all Volumes where no active pins remain.
  • Pinned Volumes are never evicted, regardless of cache pressure or LRU position.
Owner model
Owners are arbitrary strings — typically chain identifiers like "nexus" or "payments". Each chain independently pins and unpins Volumes it needs. A Volume shared by two chains (e.g., a common subtree) remains pinned as long as either chain holds a pin.

BrokerFetcher / BrokerStorer

Adapters that bridge the VolumeBroker protocol to Cashew's Fetcher and Storer protocols, enabling Merkle structures to resolve and persist through the broker layer.

}

Cashew + VolumeBroker
A BrokerFetcher backed by a VolumeBroker gives you Merkle structures that lazily load from memory (MemoryBroker) → disk (DiskBroker) → network (Ivy). A BrokerStorer backed by the same broker persists mutations as Volumes across all tiers.

Usage

let broker: any VolumeBroker = diskBroker
let fetcher = BrokerFetcher(broker: broker)
let storer = BrokerStorer(broker: broker)

// Resolve a Cashew Merkle dictionary from broker storage
let resolved = try await root.resolve(paths: paths, fetcher: fetcher)

// Persist mutations back through the broker
try newRoot.storeRecursively(storer: storer)
