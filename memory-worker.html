MemoryBroker Storage

In-memory Volume broker with LRU eviction, capacity cap, and pin-aware cache management. Pinned Volumes are never evicted regardless of cache pressure.

.package(url: "https://github.com/treehauslabs/MemoryBroker.git", from: "1.0.0")

MemoryBroker

actor VolumeBroker

public actor MemoryBroker: VolumeBroker {
    public init(
        capacity: Int? = nil
    )
}

Parameters

capacity Int? Maximum number of Volumes to hold in memory. When exceeded, the least-recently-used unpinned Volume is evicted. Nil for unbounded.

LRU Eviction

MemoryBroker uses a Least-Recently-Used eviction policy. Each fetchVolumeLocal or storeVolumeLocal call moves the accessed Volume to the head of the LRU list. When the number of stored Volumes exceeds capacity, the tail of the list is scanned for eviction candidates.

Eviction Order

  1. Start from the LRU tail (least recently accessed Volume)
  2. Skip any Volume that has at least one active, non-expired pin
  3. Evict the first unpinned Volume found
  4. Repeat until the Volume count is at or below capacity
Capacity vs. unbounded
Passing nil for capacity creates an unbounded in-memory cache with no eviction. Useful for short-lived processes or tests where memory is not a concern. In production, set a capacity to bound RSS.

Pin-Aware Eviction

MemoryBroker stores pin state as a dictionary of PinEntry values, keyed by (root, owner).

public struct PinEntry: Sendable {
    public var count: Int
    public var expiresAt: Date?
}

Pin Behavior

OperationEffect
pin(root:owner:count:ttl:) Creates or increments a PinEntry for (root, owner). If ttl is provided, sets expiresAt to now + ttl.
unpin(root:owner:count:) Decrements the pin count. If the count reaches zero, the PinEntry is removed.
unpinAll(owner:) Removes all PinEntry values for the given owner across all roots.
evictUnpinned() Removes all Volumes that have no active pins (no entries, or all entries expired or at count zero).

Eviction and Pins

A Volume is considered pinned if it has at least one PinEntry where count > 0 and either expiresAt is nil (no expiry) or expiresAt > now. Pinned Volumes are skipped during LRU eviction.

Capacity saturation
If all Volumes in the cache are pinned, eviction cannot free space. New stores will exceed the capacity cap until pins are released. Monitor the owners property and pin counts to avoid unbounded growth.

Methods

MethodBehavior
hasVolume(root:) async Check if a Volume with the given root exists in memory.
fetchVolumeLocal(root:) async Retrieve a Volume from memory. Moves it to the LRU head on access.
storeVolumeLocal(_:) async Store a Volume in memory. Triggers LRU eviction if at capacity.

Properties

PropertyTypeDescription
ownersSet<String>All owners that currently hold at least one active pin

Per-chain isolation
In a multi-chain node, each chain gets its own MemoryBroker instance. This provides per-chain LRU isolation — one chain's hot data cannot evict another chain's cache. The shared DiskBroker sits below all per-chain MemoryBrokers.
